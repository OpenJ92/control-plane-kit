## #607 Zero-Unmapped Validation

### Objects, transformation, and policies

```text
ParityManifest
x ReferenceLawOwnership
x ReferenceDemoInventory
x SuccessorEvidenceIndex
x ValidationPolicy
  -> validate_parity
    -> ParityValidationReport
```

The closed policy language has two values:

```text
foundation
  all frozen references are mapped exactly once
  mappings retain frozen identity, law, owner, and required/deferred state
  every named successor proof resolves coherently

migration-complete
  foundation laws
  + every required entry has passing proof or reviewed supersession
```

This avoids weakening the completion rule merely to validate the pre-migration
repository. The current report is valid under `foundation` and explicitly not
migration-complete:

```text
entries:             1107
required:             880
deferred:             227
incomplete required:  880
findings:                0
migration complete: false
```

### Evidence and executable laws

The evidence index is separate from the manifest. A manifest successor names a
proof identity and expected status; the evidence index contains the unique
identity, actual status, and canonical SHA-256 digest. The validator therefore
does not trust a manifest's claim that its successor passed.

Mutation-style tests prove failure for omitted mappings, stale mappings, frozen
tag/commit drift, owner/law/state drift, missing proof, status disagreement,
matching failed proof, malformed or duplicate proof records, and required work
without completion. Deferred work remains counted but does not masquerade as a
required migration success.

Five tests first failed only at the deliberate validator scaffold. The first
green attempt exposed one test expectation that named only `missing_evidence`;
the application correctly also reported `required_without_completion`. The
test was strengthened to require both independently useful facts. Review then
found and fixed a real cross-document gap: manifest tag and commit are now
compared with the authoritative inventories.

The final focused set contains 24 passing validator, manifest, test-inventory,
ownership, and demo-inventory tests. The Docker runner reads bounded JSON,
requires no network or GitHub access, writes the report atomically, and emits a
bounded summary. No raw command output, secret, environment value, credential,
or product payload enters evidence.

## #608.1 Typed Differential Language (#686)

### Objects and laws

```text
RawObservation
  = ImplementationIdentity
  x Command
  x ProcessResult
  x BoundedAuditOutput
  x StructuredBehavior
  x DeclaredArtifacts

NormalizationPolicy
  = Set IncidentalKind

compare_observations
  : RawObservation(reference)
  x RawObservation(successor)
  x NormalizationPolicy
  -> DifferentialResult
```

The four incidental kinds are closed: allocated port, generated identity,
timestamp, and container name. They are not inferred from strings, object keys,
or regular expressions. Scenario adapters must mark an incidental value using
explicit syntax:

```json
{"$incidental": {"kind": "allocated-port", "value": 49152}}
```

The interpreter type-checks that value and replaces it only when the policy
admits the same kind. Untagged values remain semantic even under a policy that
admits all four kinds. This prevents response bodies, errors, graph shape,
activity order, capabilities, observations, retention, or cleanup from being
normalized merely because their text resembles an ID, port, or timestamp.

Raw stdout and stderr are bounded base64 evidence with byte length and SHA-256
digest. Base64 is only a language-neutral transport for arbitrary captured
bytes; it is not part of the deployment core or application algebra. Process
exit, structured behavior, and declared artifacts are the comparison surface.
Nonzero completed exits are comparable behavior; timeout, output-limit, and
infrastructure failure remain distinct from behavioral drift.

### Result integrity and tests

`DifferentialResult` retains both raw observations, their pinned implementation
and source identities, normalized behavior, policy, derived differences, and
raw-observation digests. Its decoder recomputes all of those relationships, so
a stored status, digest, or normalized value cannot be altered independently.

Seven focused tests first failed only at deliberate scaffold boundaries. Eight
now pass for closed observation and result codecs, typed incidental values,
normalization idempotency, equivalent incidental values, visible semantic
drift, infrastructure failure, comparable nonzero exits, audit-only log
formatting, and tamper rejection. No frozen or application test changed.

### Handoff to #687

The effectful runner may capture and interpret these values, but it must not
invent equivalence rules. Each side executes in a separate container and emits
structured behavior. The runner must bound output, redact declared secrets
before durability, record exact commands and source identities, and derive an
evidence-index record from the closed `DifferentialResult`.
