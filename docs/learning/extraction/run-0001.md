### Evidence and executable laws

The evidence index is separate from the manifest. A manifest successor names a
proof identity and expected status; the evidence index contains the unique

does not trust a manifest's claim that its successor passed.

## #608.1 Typed Differential Language (#686)

### Objects and laws

```text
RawObservation
  = ImplementationIdentity
  x Command
  x ProcessResult
  x BoundedAuditOutput
  x StructuredBehavior
  x DeclaredArtifacts

NormalizationPolicy
  = Set IncidentalKind

compare_observations
  : RawObservation(reference)
  x RawObservation(successor)
  x NormalizationPolicy
  -> DifferentialResult
```

The four incidental kinds are closed: allocated port, generated identity,
timestamp, and container name. They are not inferred from strings, object keys,
or regular expressions. Scenario adapters must mark an incidental value using
explicit syntax:

```json
{"$incidental": {"kind": "allocated-port", "value": 49152}}
```

The interpreter type-checks that value and replaces it only when the policy
admits the same kind. Untagged values remain semantic even under a policy that
admits all four kinds. This prevents response bodies, errors, graph shape,
activity order, capabilities, observations, retention, or cleanup from being
normalized merely because their text resembles an ID, port, or timestamp.

Raw stdout and stderr are bounded base64 evidence with byte length and SHA-256
digest. Base64 is only a language-neutral transport for arbitrary captured
bytes; it is not part of the deployment core or application algebra. Process
exit, structured behavior, and declared artifacts are the comparison surface.
Nonzero completed exits are comparable behavior; timeout, output-limit, and
infrastructure failure remain distinct from behavioral drift.

### Result integrity and tests

`DifferentialResult` retains both raw observations, their pinned implementation
and source identities, normalized behavior, policy, derived differences, and
raw-observation digests. Its decoder recomputes all of those relationships, so
a stored status, digest, or normalized value cannot be altered independently.

Seven focused tests first failed only at deliberate scaffold boundaries. Eight
now pass for closed observation and result codecs, typed incidental values,
normalization idempotency, equivalent incidental values, visible semantic
drift, infrastructure failure, comparable nonzero exits, audit-only log
formatting, and tamper rejection. No frozen or application test changed.

### Handoff to #687

The effectful runner may capture and interpret these values, but it must not
invent equivalence rules. Each side executes in a separate container and emits
structured behavior. The runner must bound output, redact declared secrets
before durability, record exact commands and source identities, and derive an
evidence-index record from the closed `DifferentialResult`.

## #608.2 Isolated Differential Runner (#687)

### Objects and transformations

```text
capture_command
  : Command
  x ImplementationIdentity
  x SourceDigest
  x Timeout
  x OutputBound
  x DeclaredSecrets
  x DeclaredArtifacts
  -> RawObservation

compare CLI
  : RawObservation(reference)
  x RawObservation(successor)
  x NormalizationPolicy
  -> DifferentialResult x SuccessorEvidenceRecord
```

The runner is deliberately an interpreter for the differential language, not a
second comparison language. Subprocesses write structured behavior to the
`CPK_PARITY_BEHAVIOR_PATH` file supplied by the runner. Declared artifacts are
read explicitly, bounded, checked for declared secret material, and encoded into
the existing `RawObservation` artifact field. The command identity, source
digest, process outcome, stdout/stderr audit payloads, structured behavior, and
artifacts are all passed through the closed differential decoder before they can
be persisted.

Declared secrets are permitted only as environment material for the executed
process. Secret values are rejected from command arguments, structured behavior,
declared artifacts, and durable observations. Output redaction is also bounded:
if replacing a repeated secret with a redaction marker would expand output past
the declared output limit, the durable audit payload collapses to a bounded
redaction-limit marker, or to an empty payload when the caller's bound is even
smaller than that marker.

### Docker runner evidence

The live proof is `./differential-live-test.sh`. It builds the Docker test
image, captures a reference observation and successor observation in separate
containers, compares them in a third container, and asserts the result in a
fourth container. The scenario intentionally differs by allocated port only:

```text
reference: SCENARIO_PORT=49152
successor: SCENARIO_PORT=49153
policy:    allowed = [allocated-port]
```

The proof traverses the public runner CLI and persists JSON evidence through a
temporary Docker-mounted evidence directory. The final assertion container now
uses `docker run -i`; without `-i`, the heredoc assertion body can be lost and a
false-green live script is possible. This was caught during review before
commit and is now part of the live proof.

The Docker test image now copies the top-level extraction parity harness and
reference scripts into `/app`:

```dockerfile
COPY extraction_parity ./extraction_parity
COPY reference-inventory.sh reference-test.sh ./
```

This does not make the harness part of the installable base wheel. It only makes
the Docker test image able to execute top-level milestone tooling and inspect
the frozen-reference scripts. The base package import tests still prove that
`import control_plane_kit` remains lightweight.
