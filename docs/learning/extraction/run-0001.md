## #603 Semantic Test-Law Inventory

### Behavioral law card

```text
identity:
  extraction.reference.every-collected-test-has-one-semantic-law

law:
  every frozen unittest collection occurrence maps to exactly one stable law
  identity independent of file and TestCase location

observable result:
  collected count = 1112
  unique semantic laws = 1090
  skipped = 0
  laws with explicit subTest dimensions = 85

negative cases:
  missing or malformed test method identity
  duplicate canonical reference entries
  duplicate law identity
  ambiguous behavior name without an explicit override
  skipped test without rationale

obsolete assumptions to discard:
  unittest's optional tests-package import prefix is not semantic identity
  repeated discovery of one TestCase is not a new behavioral law

future owner:
  neutral extraction parity harness
```

### Dry run and target interface

The frozen suite contains 1,112 executions but 1,090 unique canonical test
references. Imported `TestCase` classes cause unittest to collect seven effect
dispatch laws twice and three execution-store laws six times. The inventory
therefore separates semantic identity from collection multiplicity:

```python
@dataclass(frozen=True)
class CollectedTest:
    reference: str
    law: str
    dimensions: tuple[str, ...]
    skip_reason: str | None
    collection_occurrences: int = 1
```

The optional discovery prefixes `test_x...` and `tests.test_x...` canonicalize
to the same reference before occurrences are counted. Stable law names derive
from semantic test method names, not module paths or classes. Four genuinely
ambiguous product methods use explicit reviewed qualifiers for HTTP cache,
HTTP bulkhead, idempotency gateway, and webhook Postgres behavior.

### Review finding and hardening

The first collector implementation parsed `inspect.getsource(method)`. One
architecture test intentionally contains an unindented multiline source string,
which defeated text dedenting. Collection now parses each complete source file
and locates the method AST node, matching the established architecture-policy
approach and preserving `subTest` dimensions without source rewriting.

The second collection attempt exposed duplicate unittest module identities.
Those are now explicit occurrence counts rather than duplicate laws. Nothing is
silently discarded: descriptor `count` remains 1,112 while `law_count` is 1,090.

### Validation and handoff

- Eight focused Docker tests pass.
- Frozen collection count equals the #602 execution count exactly.
- No skips exist.
- No law or canonical reference duplicates remain.
- The inventory image is removed after generation.
- #604 must classify all 1,090 laws while preserving the 1,112 occurrence
  count and the four explicit semantic overrides.
