### Review finding and hardening

The first complete run exposed one anonymous Postgres volume left by the frozen
`test.sh`. This was not hidden by normalization. The outer runner now computes
the exact before/after volume set, refuses to remove attached volumes, removes
only run-local detached additions, records those identities under
`owned_cleanup`, and takes a second inventory for final residue.

All Pottery Factory containers remained running throughout validation.
